# Архитектура Каскада Маркет

Цель: строить платформу так, чтобы один фундамент работал для сайта, PWA и будущего мобильного приложения.

## Принцип

Код делится на слои:

1. `src/lib` — общая логика: роли, маршруты, товары, финансы, статусы.
2. `src/components` — переиспользуемые UI-блоки.
3. `src/app` — страницы веб-версии.
4. Будущий `mobile` — экраны React Native / Expo.
5. Backend — Supabase: auth, database, storage, payments.

## Роли

- buyer — покупатель.
- partner — партнёр.
- seller — продавец.
- admin — администратор.

Один пользователь может иметь несколько ролей. После входа приложение должно открывать нужный режим.

## Мобильная стратегия

Этап 1: mobile-first сайт + PWA.
Этап 2: общая бизнес-логика в `src/lib`.
Этап 3: мобильное приложение на Expo / React Native.
Этап 4: общий backend и единые роли для web и mobile.

## Почему так

Если логика лежит отдельно от UI, мы сможем не переписывать маркетинг-план, роли, финансы и товары при переходе к приложению.
